# Phase 4 — Markdown twins for AI agents

**Goal:** Serve lightweight `text/markdown` twins of every motor, case study, and location so AI agents (ChatGPT browse, Perplexity, Claude, custom MCP clients) can ingest our authoritative product, location, and case-study data cheaply — without parsing full HTML, hunting JSON-LD, or running JS.

HTML stays for humans + Google. Markdown is for agents. The agent quote API stays for action.

## What gets built

### 1. New markdown twin files at build time

Generated by extending `scripts/static-prerender.mjs` (single source of truth — same script that already stamps HTML + sitemap + verification gates).

- `dist/motors/{slug}.md` — one per motor in `motor_models` where `availability='Available'` (matches the HTML prerender filter). ~29 files.
- `dist/case-studies/{slug}.md` — one per entry in `src/data/caseStudies.ts`. 5 files.
- `dist/locations/{slug}.md` — one per entry in `src/data/locations.ts`. 5 files.
- `dist/catalog.md` — single index file linking to all three sets above. Acts as the entry point for agent crawlers.

Total new artifacts: ~40 markdown files.

### 2. Markdown content schema

Every markdown file starts with a stable header so agents get business rules + canonical link before they parse details.

Common header (all files):
```markdown
---
canonical: https://www.mercuryrepower.ca/{path}
last_updated: 2026-04-25
currency: CAD
pickup_only: true
delivery_offered: false
location: Gores Landing, ON, Canada
final_quote_requires_dealer_confirmation: true
verado_status: special-order only — not in default inventory
---
```

#### Motor twin (`/motors/{slug}.md`)
Body sections, in order:
- `# {model_display}` — H1
- **Quick facts** — model, family (FourStroke/Pro XS/SeaPro/Racing), HP, model number, shaft, start type, control type
- **Pricing (CAD)** — selling price, MSRP if higher, monthly estimate at standard financing
- **Availability** — in_stock + stock count
- **Best fit for** — derived from HP band + family (e.g., "FourStroke 60–115 HP fits aluminum fishing 16–20 ft, pontoons under 22 ft")
- **Not ideal for** — explicit anti-recommendations (e.g., "Tournament bass — see Pro XS instead")
- **Build a quote** — link to `https://www.mercuryrepower.ca/motors/{slug}` and `https://www.mercuryrepower.ca/quote/motor-selection?motor={id}`
- **Agent API** — `POST https://eutsoqdpjurknjsshxes.functions.supabase.co/agent-quote-api` with sample body
- **Notes** — Verado guard, ProKicker note where applicable, financing minimum threshold

#### Case study twin (`/case-studies/{slug}.md`)
- `# {title}` — H1
- **Boat type / Region / Scenario** — three-line factbox
- **Before motor → After motor** — with HP delta
- **Recommendation** — what we sold them and why
- **Why it worked** — bullet list from `study.whyItWorked`
- **Customer quote** — blockquote from `study.customerQuote`
- **Quote a similar repower** — link to `/quote/motor-selection`
- **Related** — links to matching location markdown twins by region

#### Location twin (`/locations/{slug}.md`)
- `# {title}` — H1
- **Region / Drive time / Pickup policy** — factbox (always "Pickup at Gores Landing")
- **Common boat types** — bullet list
- **Recommended Mercury motors for this region** — top 3-5 motor markdown links
- **Relevant case studies** — markdown links filtered by `study.region === loc.region`
- **FAQs** — flat Q/A list (no JSON-LD wrapper — pure markdown)

#### Catalog index (`/catalog.md`)
- Summary of what's available, with last_updated
- Three sections: Motors / Case Studies / Locations, each as a flat list of `[title](path.md)` entries
- Agent-quote-api endpoint reference

### 3. Vercel routing — serve as `text/markdown`

Two pieces:

**a) `vercel.json` rewrites** — already serves `.md` files when present. Add explicit headers so the response advertises `Content-Type: text/markdown; charset=utf-8` and `X-Robots-Tag: noindex` (avoids duplicate indexing competing with the HTML versions in Google).

```json
{
  "source": "/(.*\\.md)",
  "headers": [
    { "key": "Content-Type", "value": "text/markdown; charset=utf-8" },
    { "key": "X-Robots-Tag", "value": "noindex, follow" },
    { "key": "Cache-Control", "value": "public, max-age=300, s-maxage=3600, stale-while-revalidate=86400" }
  ]
}
```

`noindex, follow` keeps Google from indexing the `.md` (HTML version is canonical) but still lets crawlers follow the internal `.md` → `.md` links so the graph is discoverable.

**b) No new rewrites needed** for the URL paths themselves — Vercel serves files in `dist/` directly. `/motors/{slug}.md` lands on the file at `dist/motors/{slug}.md` automatically.

### 4. Discovery surface — `llms.txt` + new `mcp.json`

**`public/llms.txt`** — append a new section near the top:

```
## Markdown twins for agents

Every motor, case study, and location has a markdown twin at the same path with `.md` appended:

- Catalog index: https://www.mercuryrepower.ca/catalog.md
- Motor: https://www.mercuryrepower.ca/motors/{slug}.md
- Case study: https://www.mercuryrepower.ca/case-studies/{slug}.md
- Location: https://www.mercuryrepower.ca/locations/{slug}.md

Markdown is the preferred ingestion format. HTML is for humans and search engines.
All prices in CAD. Pickup only at Gores Landing, ON. Final price confirmed by dealer.
```

**New `public/mcp.json`** — MCP-style discovery manifest pointing AI clients at:
- The agent-quote-api edge function
- The catalog markdown index
- IndexNow key for staleness
- Business rules block (CAD, pickup only, Verado special order)

Lightweight first version — pure JSON, no live MCP server, but conformant enough that Claude/Cursor/etc. can pick it up.

### 5. Build-time verification gates

Extend the existing `verifyErrors` block in `static-prerender.mjs`:

- `dist/catalog.md` exists, > 500 bytes, references at least one motor and one case study and one location.
- For one sample motor (first available): `dist/motors/{slug}.md` exists, starts with `---\ncanonical:`, contains `currency: CAD`, contains `pickup_only: true`, contains a quote link.
- For one sample case study (first in `caseStudies`): `dist/case-studies/{slug}.md` exists, contains the study title, contains a Mercury reference.
- For one sample location (first in `locations`): `dist/locations/{slug}.md` exists, contains "Gores Landing".
- Count parity: motor `.md` count equals motor HTML page count; case study and location `.md` counts equal data file lengths.
- **Fail the build** if any check fails. Mirrors the existing motor/case-study HTML verification.

### 6. (Stretch — only if cheap) Live verification check

After publish, fetch `https://www.mercuryrepower.ca/catalog.md` and one sample of each twin type. Confirm:
- HTTP 200
- `Content-Type: text/markdown; charset=utf-8`
- `X-Robots-Tag: noindex, follow`
- Body starts with `---` frontmatter (not HTML)

Run via the same script that does the post-publish JSON-LD audit.

## What does NOT change

- HTML prerender output — untouched. Same titles, schemas, canonicals as Phase 3.5.
- Sitemap — `.md` files are intentionally excluded from `sitemap.xml` (sitemap is for Google's HTML crawl). The `llms.txt` + `catalog.md` + per-page `.md` graph is the agent crawl path.
- React routing — agents fetch `.md` files directly off the CDN; no SPA involvement.
- Database schema — no new tables. All markdown is regenerated at build time from existing data sources.
- IndexNow — `.md` URLs are not pushed to IndexNow (Bing also doesn't want them indexed; they're agent-only).

## Acceptance criteria

- ✅ `https://www.mercuryrepower.ca/catalog.md` returns 200 with `Content-Type: text/markdown; charset=utf-8` and `X-Robots-Tag: noindex, follow`.
- ✅ Sample `https://www.mercuryrepower.ca/motors/{slug}.md` returns markdown frontmatter (not HTML), references CAD + Gores Landing, and includes the agent-quote-api endpoint.
- ✅ Sample `https://www.mercuryrepower.ca/case-studies/{slug}.md` returns markdown with the study's before/after motors and the customer quote.
- ✅ Sample `https://www.mercuryrepower.ca/locations/{slug}.md` returns markdown with FAQs and at least one related case-study link.
- ✅ `llms.txt` advertises the markdown twins.
- ✅ `mcp.json` exists at `/mcp.json` and references the catalog + agent-quote-api.
- ✅ Build fails loudly if any of the above regress.

## Open questions for you

1. **Verado in markdown twins?** Recommend: yes, generate the `.md` (so an agent that's specifically asked about Verado finds the canonical info), but the frontmatter and body explicitly state "special order only — not in default inventory; contact dealer." Alternative: skip Verado entirely from `.md` generation. **Default plan: include with explicit guard.**

2. **`mcp.json` scope?** Two options:
   - **(a)** Static manifest only — discovery hints, no live MCP server. Cheap, ships now.
   - **(b)** Add a real MCP-over-HTTP edge function later. More work, but enables agentic tool calls (search motors, get quote) over a standardized protocol.
   - **Default plan: (a) now, (b) as Phase 5.**

3. **Should `catalog.md` also list blog articles?** Currently scoped to motors/case-studies/locations only (the action-oriented surfaces). Blog is more SEO-driven and arguably belongs in `llms.txt` as it does today. **Default plan: skip blog from `catalog.md`, keep it in `llms.txt`.**
