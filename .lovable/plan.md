

# Soft Lead Capture — "Silent Save" for Anonymous Visitors

## The Philosophy
Right now you have two capture paths: Google sign-in (active save) and email form (manual save). But most visitors will do neither — they'll build a quote, maybe download the PDF, and leave. Those are your soft leads. The activity tracker already records their journey, but there's no way to reconnect a returning visitor to their previous session or quote.

The idea: **auto-save every completed quote as a soft lead** — no prompt, no friction, no contact. Just silently persist their configuration so that if they call, email, or walk in later, you can pull up exactly what they were looking at.

---

## What to Add

### 1. Auto-Save Quote Snapshot on Summary View
**File**: `src/pages/quote/QuoteSummaryPage.tsx`

When a user reaches the summary page with a configured motor, silently save a `saved_quotes` record using their anonymous session ID (already generated by the activity tracker). No email required — use `anonymous@soft-lead.local` as placeholder. This creates a linkable record tied to their `session_id` from `quote_activity_events`.

- Only fires once per session (use a ref flag)
- Saves `quote_state`, `session_id` in the record
- No toast, no UI — completely invisible to the customer
- If they later sign in via Google or save manually, the record gets updated with their `user_id` and real email (already handled by the existing save flows)

### 2. Link Activity Sessions to Saved Quotes
**File**: `src/pages/AdminQuoteActivity.tsx`

On the admin quote activity page, add a "View Quote" button on session rows that have reached the summary step. This queries `saved_quotes` by matching the `session_id` stored in the soft-lead record, letting you pull up exactly what they configured.

### 3. Admin CRM View Enhancement
**File**: `src/pages/AdminQuoteActivity.tsx`

Add a new column/badge showing lead warmth derived from behavior:
- **Hot**: Downloaded PDF or submitted deposit
- **Warm**: Reached summary page, spent 2+ minutes
- **Cool**: Selected motor but didn't finish configuring
- **Cold**: Browsed but didn't select a motor

This gives you instant context when someone calls in — "Oh, you were looking at the 115hp with the Gold package."

### 4. Returning Visitor Recognition
**File**: `src/hooks/useQuoteActivityTracker.ts`

The session ID already persists in `localStorage`, so a returning visitor gets the same session. Add a `return_visit` event type when someone comes back to the quote builder and their session already has previous events. This flags them as a warmer lead in the admin view.

---

## What Stays the Same
- No new popups, forms, or prompts for the customer
- The existing Google sign-in and email save flows are untouched
- Activity tracking continues as-is
- The customer experience is completely friction-free

The `saved_quotes` table already supports nullable `user_id` and `email`. The `quote_activity_events` table already has all needed columns. The session ID linkage is done via a shared field in both tables.
